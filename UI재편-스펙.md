# UI 재편 스펙 (진행 중)

노션을 **모티브로만** 삼아 UI를 다시 짠다. 최종 목표 인상은 **"웹앱이 아니라 프로그램(데스크톱 앱)"**.
여기서 텍스트로 하나씩 합의 → 최종 프롬프트를 Stitch에 넣어 시안 → 시안을 보고 구현. **이 문서가 사양의 원본**이고, Stitch 시안은 참고용 그림이다.

## 가져올 모티브 / 안 가져올 것

가져온다: ① 구역이 나뉜 화면 ② 항목을 끌어 옮기는 조작 ③ 세부정보의 "작업 설명" 칸(구현 완료 2026-07-22)
안 가져온다: 노션의 페이지/블록 개념, 데이터베이스 속성 편집, 사이드바 트리, 이모지 아이콘 남발, 본문을 밀어내는 상세 패널

## 확정 (사용자 결정)

1. **상세 패널은 지금 방식 유지** — 본문을 밀지 않고 오른쪽에 뜬다. 이유: 달력을 보면서 세부를 봐야 하므로 본문이 그대로 보여야 함. → 슬라이드 애니메이션만 추가(완료, 2026-07-22).
2. **작업 설명 칸** — 상세의 진행기록 위, "이 일이 무엇인지" 적는 고정 칸. 자동 저장. (완료)
3. **화면 스크롤: 창에 딱 맞추고 구역별 스크롤** — 사이드바·본문·상세가 각자 안에서만 스크롤되고 페이지 전체는 안 움직인다. "프로그램 느낌"의 핵심.
4. **입력창: 항상 보이되 한 줄로 얇게** — 인식 칩·안내문은 글을 쓸 때만 나타난다. "던지면 끝"은 유지.
5. **정보 밀도: 촘촘하게** — 글자 한 단계 줄이고 여백·간격을 좁혀 한 화면에 많이 보이게.
6. **폰도 처음부터 같이 설계** — 화면마다 PC안·폰안을 한 쌍으로 정한다.

## 미정 (다음에 정할 것)

- 색조: 지금 베이지 유지 vs 흰 배경+회색 (프로그램 느낌 방향 검토 후 결정)
- 사이드바에 무엇을 둘지 (지금은 "메모" 하나 + 하단 보관함·휴지통·백업·로그아웃)
- 상단 툴바 구성 (제목·보기 전환·검색·정렬)
- 키보드 단축키 도입 여부와 범위
- 카드/행의 모양 — 카드형 유지 vs 선으로 나뉜 행(list row)
- 보기 전환(보드·달력·표·타임라인)의 위치와 모양
- 폰 전용 규칙 (하단 탭? 상단 유지?)

## 지켜야 할 제약 (기능은 그대로)

재편은 "겉"만 바꾼다. 다음은 그대로 살아 있어야 한다:
마감형("~까지")과 ⚑ 마감 표시 / D-n 색 그라데이션 / 오늘·마감·만기 타일 / 보드의 상태 자동 판정(체크 기준) / 진행기록 체크리스트 / 달력의 기간 표시와 "이 달에 걸친 기간" / 보관함 / 휴지통 30일 / 기기 간 동기화 / 백업 / 폰에서 드래그 없이 상태 이동 / 화면 폭에 따른 배율 보정.
