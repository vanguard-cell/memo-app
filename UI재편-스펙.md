# UI 재편 스펙 (진행 중)

노션을 **모티브로만** 삼아 UI를 다시 짠다. 최종 목표 인상은 **"웹앱이 아니라 프로그램(데스크톱 앱)"**.
여기서 텍스트로 하나씩 합의 → 최종 프롬프트를 Stitch에 넣어 시안 → 시안을 보고 구현. **이 문서가 사양의 원본**이고, Stitch 시안은 참고용 그림이다.

## 가져올 모티브 / 안 가져올 것

가져온다: ① 구역이 나뉜 화면 ② 항목을 끌어 옮기는 조작 ③ 세부정보의 "작업 설명" 칸(구현 완료 2026-07-22)
안 가져온다: 노션의 페이지/블록 개념, 데이터베이스 속성 편집, 사이드바 트리, 이모지 아이콘 남발, 본문을 밀어내는 상세 패널

## 확정 (사용자 결정)

1. **상세 패널은 지금 방식 유지** — 본문을 밀지 않고 오른쪽에 뜬다. 이유: 달력을 보면서 세부를 봐야 하므로 본문이 그대로 보여야 함. → 슬라이드 애니메이션만 추가(완료, 2026-07-22).
2. **작업 설명 칸** — 상세의 진행기록 위, "이 일이 무엇인지" 적는 고정 칸. 자동 저장. (완료)
3. **상세 닫기: 빈 곳 클릭 · Esc** — 오른쪽으로 미끄러져 나가며 닫힌다(완료). 다른 메모를 누르면 닫히지 않고 교체. 입력 중 Esc는 입력 취소 우선. (프로그램 느낌 요소 ⑦ 키보드의 첫걸음)
4. **화면 스크롤: 창에 딱 맞추고 구역별 스크롤** — 사이드바·본문·상세가 각자 안에서만 스크롤되고 페이지 전체는 안 움직인다. "프로그램 느낌"의 핵심.
5. **입력창: 항상 보이되 한 줄로 얇게** — 인식 칩·안내문은 글을 쓸 때만 나타난다. "던지면 끝"은 유지.
6. **정보 밀도: 촘촘하게** — 글자 한 단계 줄이고 여백·간격을 좁혀 한 화면에 많이 보이게.
7. **폰도 처음부터 같이 설계** — 화면마다 PC안·폰안을 한 쌍으로 정한다.

8. **PC 달력 = 화면 좌우 절반 고정** (2026-07-24 완료) — 왼쪽 절반: 메뉴+달력(고정, 안 밀림), 오른쪽 절반: 목록+상세를 **나란히**. 날짜 클릭→오른쪽에 목록, 목록 항목 클릭→그 오른쪽에 상세. 빈 곳 클릭·Esc로 상세→목록→닫힘. **비율(1fr:1fr) 기반이라 화면 크기가 달라져도 좌우 배분이 안 변한다(사용자 요구)**. cal-split은 grid 2열, 달력 보기에서만 본문·입력창을 :has()로 전체폭 확장. **이 "좌우 절반 · 본문 고정 · 우측에서 파고드는 목록→상세" 패턴이 재편의 기준** — 보드·표에도 확장할지 검토.

9. **위쪽 툴바를 왼쪽 절반(달력 폭)에 정렬** (2026-07-24 완료, 사용자 확정) — 입력창·툴바를 달력 왼쪽 컬럼과 동일 폭(calc(50% - 12px))으로 맞춰 왼쪽 절반에 모으고, 오른쪽 절반은 목록·상세 전용으로 비웠다. 달력 보기에선 툴바 nowrap + 검색 max 300px로 타일·검색·보기전환이 한 줄. 입력창·툴바·달력 좌우 끝이 픽셀 단위로 정렬됨(1440·1920·2560 확인). **이제 "왼쪽 절반=메뉴+입력+툴바+달력 / 오른쪽 절반=목록+상세"가 완성형** — 이 좌우 분할이 재편의 골격.

10. **위쪽 줄 재편 + 타일 제거** (2026-07-24 완료, 사용자 지시) — 오늘/마감·만기 타일 삭제(시간 필터 기능 통째로 제거), "이 달에 걸친 기간" 칩 줄 삭제. 새 위쪽: **던지기 입력(왼쪽) + 보기전환(오른쪽 위) 한 줄 → 검색 한 줄**. InputBar를 App에서 MemosView 상단(mv-head)으로 옮김. 폰(<900px)은 세로 스택. 급한 순서는 보드 정렬·D-n 색이 대신하므로 타일 없이도 됨. **타일 재도입 금지.**

11. **던지기 입력 제거 → 보드 칸별 + 로 작성** (2026-07-24 완료, 사용자 지시) — 상단 "여기에 그냥 던지세요" 입력 삭제. 보드 각 칸(할일/진행중/완료) 헤더에 **+**, 누르면 오른쪽에 **작성 패널**(ComposePanel = 상세와 같은 자리, 안에 InputBar 재사용). 저장 시 그 칸 상태로 생성(진행중=stage, 완료=완료처리) 후 방금 만든 메모 상세로 이어짐. 폰은 + → 전체 오버레이. 상단은 이제 **검색 + 보기전환** 한 줄. **보관함 생성은 작성 패널의 "보관함에 넣기"가 유일한 통로**(던지기 사라져서). 표·타임라인엔 추가 입력 없음(보드·달력에서). InputBar는 status/onSaved props로 작성 패널 겸용.

12. **작성 = 상세 패널로 통합** (2026-07-24 완료, 사용자 지시) — 별도 ComposePanel(InputBar 재사용) 폐기. 보드 칸의 + → **그 칸 상태의 초안 메모(제목 빈칸·오늘 예정)를 addMemo로 바로 만들고 상세를 연다**. 상세 = 작성: 제목 인라인 편집(빈 초안이면 자동 포커스), 예정일 인라인 date input(기본 오늘, D-day 옆에), 작업설명·진행기록 동일. 빈 초안(제목·기록·설명 다 빔)은 패널 닫을 때 **purgeMemos로 완전 삭제**(휴지통 안 남김) — discardIfEmptyDraft는 getMemos()로 최신 상태 읽음. ComposePanel.jsx·InputBar.jsx **삭제됨**(던지기 입력 완전 폐기). 제목 인라인 편집은 모든 메모에 적용(정보 수정은 날짜·기간 편집용으로 유지).

## 미정 (다음에 정할 것)

- 색조: 지금 베이지 유지 vs 흰 배경+회색 (프로그램 느낌 방향 검토 후 결정)
- 사이드바에 무엇을 둘지 (지금은 "메모" 하나 + 하단 보관함·휴지통·백업·로그아웃)
- 상단 툴바 구성 (제목·보기 전환·검색·정렬)
- 키보드 단축키 도입 여부와 범위
- 카드/행의 모양 — 카드형 유지 vs 선으로 나뉜 행(list row)
- 보기 전환(보드·달력·표·타임라인)의 위치와 모양
- 폰 전용 규칙 (하단 탭? 상단 유지?)

## 지켜야 할 제약 (기능은 그대로)

재편은 "겉"만 바꾼다. 다음은 그대로 살아 있어야 한다:
마감형("~까지")과 ⚑ 마감 표시 / D-n 색 그라데이션 / 오늘·마감·만기 타일 / 보드의 상태 자동 판정(체크 기준) / 진행기록 체크리스트 / 달력의 기간 표시와 "이 달에 걸친 기간" / 보관함 / 휴지통 30일 / 기기 간 동기화 / 백업 / 폰에서 드래그 없이 상태 이동 / 화면 폭에 따른 배율 보정.
