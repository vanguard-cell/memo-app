# 「내 기록」 사용설명서

## 이 앱의 목적

머리에 담아두지 말고 전부 여기에 던지기 위한 앱. 던지는 순간부터 시스템이 대신 기억하고, 해야 할 일은 완료할 때까지 나를 괴롭히고, 끝난 일도 버리지 않고 보존해서 몇 년 뒤에도 꺼내볼 수 있게 한다.

**원칙: 심플함.** 화면 하나, 입력창 하나. 위에는 요약 타일(괴롭히기), 아래는 보드·달력·표·타임라인(관리). 기능을 늘릴 때도 화면·버튼·개념을 늘리지 않는 방향을 우선한다.

## 1. 던지기 — 맨 위 입력창

어느 화면에서든 맨 위 입력창에 한 줄로 그냥 쓴다. 형식 없음.

예: "A업체 계약 26.5.30~27.5.29" / "7/6 이사" / "내일 세금계산서 확인"

자동 인식되는 것:

| 인식 대상 | 예시 |
|---|---|
| 기간 (~) | 26.5.30~27.5.29, 7/3~7/5 |
| 날짜 | 7.6, 7/6, 26.7.6, 7월 6일 |
| 상대 날짜 | 오늘, 내일, 모레, 다음주 |

- 인식된 건 입력창 아래 칩으로 표시. 맞으면 Enter로 저장, 틀리면 칩의 ×로 제거.
- 저장 전 확인을 거치는 이유: 잘못된 날짜로 알림이 오면 시스템을 못 믿게 되기 때문.
- 날짜는 기한 칸으로 빠지고 제목에는 내용만 남는다 ("7/6 이사" → 제목 "이사", 기한 7/6).
- **연도 없이 쓴 날짜가 이미 지난 날이면 내년으로 해석** — 여름에 "1/20 보안교육"을 던지면 내년 1월 20일. (지난 날짜로 적고 싶으면 연도까지 쓰기: "26.1.20")
- **마감형 — "~까지"**: "7/30까지 보고서"처럼 쓰면 그날 하는 일이 아니라 **그날까지 끝낼 일**로 인식. 던진 날부터 계속 보이고, 카드 배지에 **"7.30까지 D-n"** — 어느 날이 마감인지 날짜가 같이 보인다 (2026-07-19). 달력에는 **마감일에만 빨간 "마감" 조각 하나** + 위 "이 달에 걸친 기간" 줄에 "~7.30 마감" 칩 (예전엔 시작·중간 조각까지 그려져 한 메모가 여러 개처럼 겹쳐 보였음 — 정리됨). "마감·만기" 타일을 누르면 마감 가까운 순으로 모아 볼 수 있다.
- 타일 필터(오늘/마감·만기)는 보드·표·타임라인에만 적용된다 — **달력은 항상 전체**를 보여준다.
- **날짜 없이 던지면 자동으로 오늘 기한** — 바로 "오늘 할 일"에 들어가고, 안 하면 다음날부터 "N일째"로 괴롭힌다.
- **다른 날짜에 넣고 싶으면**: 글에 날짜를 쓰거나("7/20 견적 회신"), 칩 줄의 **"날짜 지정" 버튼**으로 달력에서 고른다 — 고른 날짜가 "기한" 칩으로 붙고, 틀리면 ×. 글에 쓴 날짜보다 버튼으로 고른 날짜가 우선.
- **"보관함에 넣기" 버튼**: 할 일이 아니라 **알아둘 것**(연락처, 모델명, 비밀번호 힌트 등)을 넣는 곳. 기한 없이 저장되어 오늘·달력에 절대 안 뜨고, 메모탭 검색으로 꺼내본다. 검색 결과에 보라색 "보관" 배지로 구분되어 보인다. 다시 할 일로 만들려면 메모를 열어 "오늘 할 일로 꺼내기". ⚠️ 비밀번호 원문은 넣지 말 것 — 서버·백업 파일에 평문으로 저장된다. 본인만 아는 힌트만. (주간 기간으로 넣고 싶으면 "7/14~7/18"처럼 기간을 직접 쓰면 된다)

## 2. (제거됨) 업체 인식·이어붙이기

업체라는 개념은 앱에 두지 않는다 (2026-07-15 결정). "라온"이 업체라는 건 내가 아는 것이고, 앱에서는 **검색어**일 뿐 — 검색하면 계약·수리·방문 등 모든 이력이 나오고 그걸로 후속 업무를 진행한다. 같은 건의 기록은 기존 메모를 열어 진행기록에 이어 쓴다.

## 3. 요약 타일 — 괴롭히는 곳 (오늘 탭 대체, 2026-07-15)

오늘 탭은 없어지고 화면이 하나로 합쳐졌다. 대신 화면 맨 위에 **요약 타일 두 개**가 항상 떠 있다:

- **오늘 n** — 오늘까지 해야 하는 것. **밀린 게 있으면 타일이 빨갛게 변하며 "· 밀림 n"이 병기**된다 (예전 빨간 배지의 역할). 평소엔 조용하고 문제 있을 때만 소리친다.
- **만기 n** — 계약 만기 60일 안(없으면 0으로 상시 표시). **누르면 가장 가까운 만기부터 순서대로** 보인다 (표 보기에서는 상태 구분 없이 날짜순으로 쫙).
- **타일을 누르면 그 조건으로 걸러진다** — 보드·달력·표·타임라인 어느 보기에서든. 다시 누르면 해제.
- **미루기는 메모를 눌러 여는 패널에서**: 완료 처리 · 내일로 · 날짜로 (2026-07-16 카드의 버튼들을 패널로 통합 — 카드는 제목·배지만). 내일로 = 기한이 실제로 내일로 이동(달력도 함께), 날짜로 = 고른 날짜로. **기한이 이미 미래인 메모에서는 버튼이 "+1일"로 바뀌어 기한을 하루 뒤로 민다** (내일이 기한인 메모에서 "내일로"가 아무 일도 안 하던 것 개선, 2026-07-21). 계약 만기 알림만은 만기일을 건드리지 않고 그 날짜까지 숨김. 상태 바꾸기는 체크(자동)나 보드 드래그로 — 시작/할일로 버튼은 없앴다.
- **완료 되돌리기**: 완료를 누르면 화면 아래에 "'○○' 완료 — 되돌리기"가 몇 초간 뜬다. (놓쳤으면 완료 열 → 다시 열기)
- **폰은 접속하면 "오늘" 타일이 켜진 채 시작** — 예전 오늘 탭과 같은 첫 화면.
- 기간 메모에 오늘 날짜의 진행기록 줄이 있으면 보드 카드에 그 줄이 보인다 (예: 오늘의 식단).

PC 레이아웃: 상세 패널은 오른쪽에 뜨고, **패널이 열리면 본문 폭이 자동으로 줄어 가려지지 않는다** (화면이 충분히 넓으면 그대로). 로그아웃·백업은 사이드바 맨 아래.

의도: "A업체 계약 26.5.30~27.5.29"라고 메모만 해두면 내년 봄에 만기 타일에 1이 켜진다. 내가 기억하는 게 아니라 시스템이 기억한다.

## 4. 진행 기록 — 과정을 남기는 곳

메모를 누르면 상세가 열린다 (팝업 없음). **PC는 오른쪽 패널**로 붙어 달력·목록을 보면서 동시에 기록하고, **폰은 누른 줄이 그 자리에서 상세로 바뀐다** (2026-07-20 — 예전엔 줄 아래 새 창이 붙어 제목이 두 번 보였는데, 이제 줄 자체가 펼쳐진다). 접을 때는 상세 오른쪽 위 **▴** 버튼.

**폰의 상태 이동 (2026-07-18)**: PC는 보드에서 카드를 드래그해 할일↔진행중↔완료를 옮기지만 폰은 드래그가 안 되므로, 폰 상세 맨 위에 **상태: [할일] [진행중] [완료]** 버튼이 있다 — 누르면 보드 열이 바로 바뀐다. (폰에서는 이 버튼이 완료 처리·다시 열기 역할까지 하므로 별도 완료 버튼은 없다)

**폰의 보드 (2026-07-19)**: PC처럼 **할일|진행중 두 열이 나란히** 보인다 — 뭐가 밀려 있고 뭐가 굴러가는 중인지 한 화면에서 파악. **완료는 아래 "완료 N건 펼치기" 버튼** 뒤에 접혀 있다. 카드를 누르면 상세가 그 줄 아래 **전체 폭**으로 펼쳐진다.

- 한 줄씩 추가하면 날짜와 함께 쌓임
- 줄마다 체크박스 — 그 자리에서 하나씩 완료 체크, 진행률(체크 1/3)이 목록에도 표시
- 상태 자동 전환은 **체크 기준**: 줄만 적어둔 건(계획) 할일 그대로, 체크박스를 하나라도 체크하면(착수) 진행중
- 수정이 아니라 누적 — 지난 과정이 지워지지 않는 게 핵심
- 자동 로그 없음. 기록엔 내가 쓴 내용만
- 잘못 적었을 때: 줄의 글자를 누르면 그 자리에서 날짜·내용 수정 (Enter 저장, Esc 취소). 줄에 마우스를 올리면 나오는 ×로 삭제
- ~~파일 첨부~~ — 2026-07-16 제거. 한 번도 안 쓰였고(증빙 파일은 회사 폴더 보관 방침), 자리만 차지해서 삭제. 파일이 필요한 메모에는 진행기록에 "어느 폴더에 있는지"를 적는다.

## 5. 달력 — 모든 것이 날짜로 연결 (메모탭의 '달력' 보기)

2026-07-15부터 달력은 별도 탭이 아니라 **메모탭 상단 토글(보드/달력/표/타임라인)의 두 번째 보기**다. 검색이 달력에도 적용된다 — "라온"을 검색하면 라온 관련 일정만 있는 달력이 된다. 검색 중에는 달력 위에 노란 안내줄이 떠서 "걸러진 상태"임을 알려준다.

모든 메모가 자기 날짜에 나타난다:

- 기한 있는 메모 → 그 날짜
- 기간(~) 메모 → 시작일부터 만기일까지 전부 (시작/만기 표시)

규칙: 메모는 자기 날짜에만 표시되고, 진행 기록(체크리스트)은 메모 안에서만 산다. 7/6 "이사"의 체크리스트는 7/6 메모를 열면 있다 — 적은 날짜에 흩어지지 않는다.

**예외 — 기간 메모의 날짜별 기록**: 기간 메모의 진행기록에 그 날짜 줄이 있으면 달력 칸에 제목 대신 그 줄이 보인다. 주간 식당 메뉴에 딱 좋다: "식당 메뉴"를 이번 주 기간으로 저장하고 진행기록에 날짜별로 "월: 제육볶음…" 5줄을 넣으면, 월요일 칸엔 월요일 메뉴가 그대로 보인다. 오늘 화면에도 오늘 줄이 같이 표시된다.

**기간 메모의 달력 표시 (2026-07-19 보완)**: 시작일엔 초록 "시작", 만기일엔 "만기"(마감형은 "마감"), 중간 날짜엔 연한 보라 조각. **달력 맨 위 "이 달에 걸친 기간" 줄**에 이 달과 겹치는 기간 메모가 보라 칩으로 모두 나온다 — 1년짜리 계약처럼 **한 달을 넘는 기간은 달력 칸에 안 그려지므로**(도배 방지) 여기서 확인하고, 칩을 누르면 상세가 열린다. 날짜를 누르면 그날 목록에 그 날짜에 걸친 기간이 "기간"·"기간 중 (n일차)"로 전부 나온다 — 폰에서 보라 막대가 뭔지 여기서 확인.

달력에서 **오늘은 날짜 숫자의 파란 동그라미**, **클릭해서 선택한 날은 파란 테두리** — 둘이 같이 보여도 다중선택이 아니다. **완료된 메모의 칩은 회색·취소선**으로 표시되어 남은 일과 구분된다. 항목 위에 마우스를 올리면 그 항목만 테두리가 켜지고(누르면 상세), 빈 곳에 올리면 칸이 살짝 어두워진다(누르면 그날 전체 목록).

날짜 클릭 → 그날 내용이 아래 펼쳐지고, 그 자리에서 바로 새 메모 작성 가능(클릭한 날짜가 자동 기한). 한 칸에 4개 초과는 +n으로 접힘.

**폰에서는 달력이 다르게 보인다 (2026-07-19 확정)**: 칸이 좁아 글자가 안 읽히므로 칸에는 **색 막대만** 표시된다(어느 날에 뭐가 몇 개 있는지 한눈에). 달력 바로 아래에는 **선택한 날짜 하나의 목록만** 붙는다 — 열 때부터 오늘이 선택돼 있고, **다른 날짜를 누르면 화면 이동 없이 그 자리에서 내용만 바뀐다** (아이폰 기본 달력과 같은 조작감). 한때 달 전체를 아래에 나열하고 누르면 스크롤 이동하는 방식을 썼으나 조작감이 나빠서 뺐다.

**드래그로 우선순위**: 날짜를 눌러 펼쳐진 목록에서 항목을 끌어 순서를 바꾸면 그날의 우선순위가 된다. 이 순서는 저장되고, 달력 칸 안의 칩 순서와 오늘 화면의 "오늘 할 일" 순서에도 그대로 반영된다.

**드래그로 이동**: 달력의 메모를 끌어서 다른 날짜에 놓으면 날짜가 바뀐다.
- 기한 메모 → 기한이 놓은 날짜로
- 날짜 없는 메모 → 놓은 날짜가 기한이 됨
- 기간 메모 → 시작/만기 조각을 끌면 그쪽 끝만, 중간 조각을 끌면 기간 전체가 이동
- (드래그는 PC 전용. **폰에서는 날짜를 눌러 펼친 목록에서 줄마다 "이동" 버튼** — 달력이 떠서 날짜를 고르면 같은 규칙으로 이동)

의도: 달력만 넘겨봐도 하루하루가 재구성되는 것.

## 6. 메모 — 작업 트래커 (보드 · 달력 · 표 · 타임라인)

노션 작업 트래커처럼 쓴다. 상단 토글로 네 가지 보기를 전환하고, **기본은 보드**. 마지막에 보던 보기를 기억한다 — 단 **폰은 들어올 때 항상 보드부터** 시작한다 (2026-07-19). 달력 보기는 5장 참고.

- **보드** — 할일 · 진행중 · 완료 3열. 카드를 옆 열로 끌면 상태가 바뀌고 (완료 열로 끌면 완료 처리), **같은 열 안에서 끌면 순서가 바뀐다** (저장됨). 폰에서는 카드를 누르면 카드 자리가 상세로 펼쳐진다(▴로 접기). 기본 정렬은 급한 순 — 밀린 것 → 오늘 → D-n. 카드에 기한 배지("3일째"·"오늘"·D-n)와 체크 진행률, 업체 칩. **D-n 배지 색은 가까울수록 따뜻하고 멀수록 차갑다 (2026-07-19)** — 밀림 빨강 → 오늘 노랑 → D-1 주황 → D-3 노랑 → D-5 연두 → D-7 초록 → D-10 청록 → D-14부터 파랑. 하루 차이도 색으로 구분되어 훑기만 해도 급한 순서가 보인다 (보드·표 공통). **진행중 자동 판정은 체크 기준** — 줄만 적어두면 할일에 남고, 체크를 하나라도 하면 진행중으로 넘어간다 (체크 없이 시작한 건 "시작 ›" 버튼이나 드래그로). 완료 열은 **최근 완료한 순**으로 8건만, 나머지는 표에서 (표·타임라인의 완료도 같은 순서).
- **표** — 전체 목록. 상태 / 제목 / 기한·기간 / 체크 / 작성일. 보관·완료 포함 전부 나온다.
- **타임라인** — 월 단위 간트. 상태 규칙과 같은 철학("줄 적음 = 계획, 체크 = 착수")으로 그린다: **할일(착수 전)은 줄이 있어도 현재 기한 자리에 점만** — 등록일로 소급하지 않고, 내일로/날짜로로 옮기면 점이 따라간다. **진행중(착수 후)은 첫 기록부터 기한·오늘 중 늦은 쪽까지 막대** — 오래 끄는 건일수록 길어진다. 완료는 활동 구간~완료일, 기간(~) 메모는 기간 그대로. **진행중 → 할일 순으로 묶여 있고 완료는 접혀 있다**("완료 n건 펼치기"). 제목 앞 상태 점(막대와 같은 색), 제목 밑에 구간·체크 요약("7.8 ~ 진행중 · 체크 4/7"). 파란 세로줄 = 오늘. ‹ ›로 달 이동. **막대에 마우스를 올리면 진행기록(최근 8줄·체크 상태)이 카드로 떠서** 클릭 없이 내용을 훑을 수 있다 — 누르면 패널로 전체 보기.
- **검색은 어느 보기에서든 동일** — 제목·진행기록, 보관·완료까지 통합 검색. 보드에서 검색하면 걸린 보관 메모는 아래에 따로 표시되고, 표에서는 일치한 기록 줄이 날짜와 함께 표시.
- **순서도 어느 보기에서든 동일** — 급한 순(밀림 → 오늘 → D-n)이 항상 먼저라 새로 던진 메모는 자기 날짜 자리로 바로 올라온다. 보드에서 드래그한 순서는 **같은 D-day끼리의 순서**만 가르고, 표·타임라인에도 그대로 적용.
- **백업 버튼** (PC는 사이드바 맨 아래 로그아웃 옆, 폰은 상단) — 메모·점검·순서 전체를 JSON 파일로 내려받는다. 한 달에 한 번쯤 눌러서 파일을 보관해두면 계정 문제·실수 삭제에 대비한 보험이 된다 (이 파일로 복원 가능)

의도: "25년 ○○업체 계약건 내용 가져와줘" — 몇 년이 지나도 과정 전체가 그대로 나와야 한다. 판 전체를 조망할 땐 보드, 뒤질 땐 표, 일정 감각은 타임라인.
