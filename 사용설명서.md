# 「내 기록」 사용설명서

## 이 앱의 목적

머리에 담아두지 말고 전부 여기에 던지기 위한 앱. 던지는 순간부터 시스템이 대신 기억하고, 해야 할 일은 완료할 때까지 나를 괴롭히고, 끝난 일도 버리지 않고 보존해서 몇 년 뒤에도 꺼내볼 수 있게 한다.

**원칙: 심플함.** 화면은 탭 3개(오늘·달력·메모), 입력창 하나. 기능을 늘릴 때도 탭·버튼·개념을 늘리지 않는 방향을 우선한다.

---

## 1. 던지기 — 맨 위 입력창

어느 화면에서든 맨 위 입력창에 한 줄로 그냥 쓴다. 형식 없음.

예: "A업체 계약 26.5.30~27.5.29" / "7/6 이사" / "내일 세금계산서 확인"

자동 인식되는 것:

| 인식 대상 | 예시 |
|---|---|
| 기간 (~) | 26.5.30~27.5.29, 7/3~7/5 |
| 날짜 | 7.6, 7/6, 26.7.6, 7월 6일 |
| 상대 날짜 | 오늘, 내일, 모레, 다음주 |
| 업체 | "○○업체"라고 쓰거나, 전에 등장한 업체명 |

- 인식된 건 입력창 아래 칩으로 표시. 맞으면 Enter로 저장, 틀리면 칩의 ×로 제거.
- 저장 전 확인을 거치는 이유: 잘못된 날짜로 알림이 오면 시스템을 못 믿게 되기 때문.
- 날짜는 기한 칸으로 빠지고 제목에는 내용만 남는다 ("7/6 이사" → 제목 "이사", 기한 7/6).
- **연도 없이 쓴 날짜가 이미 지난 날이면 내년으로 해석** — 여름에 "1/20 보안교육"을 던지면 내년 1월 20일. (지난 날짜로 적고 싶으면 연도까지 쓰기: "26.1.20")
- **날짜 없이 던지면 자동으로 오늘 기한** — 바로 "오늘 할 일"에 들어가고, 안 하면 다음날부터 "N일째"로 괴롭힌다. 다른 날짜가 필요하면 글에 쓰거나 메모를 열어 지정.
- **"보관함에 넣기" 버튼**: 할 일이 아니라 **알아둘 것**(연락처, 모델명, 비밀번호 힌트 등)을 넣는 곳. 기한 없이 저장되어 오늘·달력에 절대 안 뜨고, 메모탭 검색으로 꺼내본다. 검색 결과에 보라색 "보관" 배지로 구분되어 보인다. 다시 할 일로 만들려면 메모를 열어 "오늘 할 일로 꺼내기". ⚠️ 비밀번호 원문은 넣지 말 것 — 서버·백업 파일에 평문으로 저장된다. 본인만 아는 힌트만. (주간 기간으로 넣고 싶으면 "7/14~7/18"처럼 기간을 직접 쓰면 된다)

## 2. 이어붙이기 — 메모 쪼개짐 방지

같은 업체의 진행중 메모가 있으면 "기존 메모에 이어붙일까요?"를 먼저 제안한다.
Enter = 이어붙이기, Shift+Enter = 새 메모.

의도: 하나의 사건은 하나의 메모 안에서 처음부터 끝까지. "견적 → 비교 → 선택 → 계약" 과정이 한눈에 보이도록, 기록이 흩어지는 걸 입력 단계에서 차단.

## 3. 오늘 — 괴롭히는 곳

접속하면 처음 보이는 화면. 탭(PC는 왼쪽 사이드바, 폰은 상단)에 빨간 숫자로 밀린 개수 표시. PC에서 상세 패널은 오른쪽에 떠서 본문을 밀지 않는다 — 탭을 오가거나 패널을 열어도 화면이 흔들리지 않음. 로그아웃은 사이드바 맨 아래.

- 미루고 있는 일 — 기한 지난 것. "3일째"처럼 며칠 미뤘는지 표시, 완료 전까지 사라지지 않음
- 오늘 할 일 — 오늘이 기한인 것
- 다가오는 일정·만기 — 기한 7일 전부터, 계약 만기는 60일 전부터 미리 등장
- 이번 달 점검 — 점검 탭의 이번 달 시행 업무 중 안 한 것. 완료 버튼은 점검 표와 같은 데이터(양쪽에 바로 반영), 줄을 누르면 점검 표로 이동

각 줄에서 바로 완료 / 내일로 / 날짜로 가능. **내일로 = 기한이 실제로 내일로 이동** — 달력에서도 내일 칸으로 옮겨진다. **날짜로 = 달력에서 고른 날짜로 기한 이동** (모든 섹션에서 사용 가능). (계약 만기 알림만은 만기일을 건드리지 않고 그 날짜까지 숨김 — "다가오는 만기" 알림이 너무 일찍부터 보이면 날짜로를 눌러 "만기 2주 전에 다시 보여줘" 식으로 미룰 수 있다)

**완료 되돌리기**: 완료를 누르면 화면 아래에 "'○○' 완료 — 되돌리기"가 몇 초간 뜬다. 실수로 눌렀으면 바로 되돌리기. (놓쳤으면 메모탭 → 완료 필터 → 다시 열기)

**드래그로 우선순위**: "오늘 할 일", "다가오는 일정·만기", "이번 달 점검"에서 항목을 끌어 순서를 바꿀 수 있다. 같은 날짜(같은 D-day)끼리만 바뀌며, 이 순서는 저장된다. (드래그는 PC 전용)

의도: "A업체 계약 26.5.30~27.5.29"라고 메모만 해두면 내년 봄에 시스템이 알아서 "만기 D-60"을 올린다. 내가 기억하는 게 아니라 시스템이 기억한다.

## 4. 진행 기록 — 과정을 남기는 곳

메모를 누르면 상세가 열린다 (팝업 없음). **PC는 오른쪽 패널**로 붙어 달력·목록을 보면서 동시에 기록하고, **폰은 누른 줄 바로 아래에 펼쳐진다** — 새 창이 덮지 않아 목록 흐름이 안 끊긴다. 점검 업무도 똑같이 동작.

- 한 줄씩 추가하면 날짜와 함께 쌓임
- 줄마다 체크박스 — 그 자리에서 하나씩 완료 체크, 진행률(체크 1/3)이 목록에도 표시
- 상태 자동 전환은 **체크 기준**: 줄만 적어둔 건(계획) 할일 그대로, 체크박스를 하나라도 체크하면(착수) 진행중
- 수정이 아니라 누적 — 지난 과정이 지워지지 않는 게 핵심
- 자동 로그 없음. 기록엔 내가 쓴 내용만
- 잘못 적었을 때: 줄의 글자를 누르면 그 자리에서 날짜·내용 수정 (Enter 저장, Esc 취소). 줄에 마우스를 올리면 나오는 ×로 삭제
- **파일 첨부**: 패널 맨 아래 "파일" 칸에 계약서·견적서 등을 끌어다 놓거나 [+ 올리기]. 파일명을 누르면 열리고, 어느 기기에서든 보인다 (클라우드 저장, 무료 1GB)

## 5. 달력 — 모든 것이 날짜로 연결

모든 메모가 자기 날짜에 나타난다:

- 기한 있는 메모 → 그 날짜
- 기간(~) 메모 → 시작일부터 만기일까지 전부 (시작/만기 표시)

규칙: 메모는 자기 날짜에만 표시되고, 진행 기록(체크리스트)은 메모 안에서만 산다. 7/6 "이사"의 체크리스트는 7/6 메모를 열면 있다 — 적은 날짜에 흩어지지 않는다.

**예외 — 기간 메모의 날짜별 기록**: 기간 메모의 진행기록에 그 날짜 줄이 있으면 달력 칸에 제목 대신 그 줄이 보인다. 주간 식당 메뉴에 딱 좋다: "식당 메뉴"를 이번 주 기간으로 저장하고 진행기록에 날짜별로 "월: 제육볶음…" 5줄을 넣으면, 월요일 칸엔 월요일 메뉴가 그대로 보인다. 오늘 화면에도 오늘 줄이 같이 표시된다.

달력에서 **오늘은 파란 바탕**, **클릭해서 선택한 날은 파란 테두리** — 둘이 같이 보여도 다중선택이 아니다.

날짜 클릭 → 그날 내용이 아래 펼쳐지고, 그 자리에서 바로 새 메모 작성 가능(클릭한 날짜가 자동 기한). 한 칸에 3개 초과는 +n으로 접힘.

**드래그로 우선순위**: 날짜를 눌러 펼쳐진 목록에서 항목을 끌어 순서를 바꾸면 그날의 우선순위가 된다. 이 순서는 저장되고, 달력 칸 안의 칩 순서와 오늘 화면의 "오늘 할 일" 순서에도 그대로 반영된다.

**드래그로 이동**: 달력의 메모를 끌어서 다른 날짜에 놓으면 날짜가 바뀐다.
- 기한 메모 → 기한이 놓은 날짜로
- 날짜 없는 메모 → 놓은 날짜가 기한이 됨
- 기간 메모 → 시작/만기 조각을 끌면 그쪽 끝만, 중간 조각을 끌면 기간 전체가 이동
- (드래그는 PC 전용. **폰에서는 날짜를 눌러 펼친 목록에서 줄마다 "이동" 버튼** — 달력이 떠서 날짜를 고르면 같은 규칙으로 이동)

의도: 달력만 넘겨봐도 하루하루가 재구성되는 것.

## 6. 메모 — 작업 트래커 (보드 · 표 · 타임라인)

노션 작업 트래커처럼 쓴다. 상단 토글로 세 가지 보기를 전환하고, **기본은 보드**. 마지막에 보던 보기를 기억한다.

- **보드** — 할일 · 진행중 · 완료 3열. 카드를 옆 열로 끌면 상태가 바뀌고 (완료 열로 끌면 완료 처리), **같은 열 안에서 끌면 순서가 바뀐다** (저장됨). 폰에서는 카드의 시작/할일로/완료 버튼으로. 기본 정렬은 급한 순 — 밀린 것 → 오늘 → D-n. 카드에 기한 배지("3일째"·"오늘"·D-n)와 체크 진행률, 업체 칩. **진행중 자동 판정은 체크 기준** — 줄만 적어두면 할일에 남고, 체크를 하나라도 하면 진행중으로 넘어간다 (체크 없이 시작한 건 "시작 ›" 버튼이나 드래그로). 완료 열은 최근 8건만, 나머지는 표에서.
- **표** — 전체 목록. 상태 / 제목 / 업체 / 기한·기간 / 체크 / 작성일. 보관·완료 포함 전부 나온다.
- **타임라인** — 월 단위 간트. **진행기록이 쌓인 메모는 실제 활동 구간이 막대로** 그려진다: 첫 기록 날짜부터 마지막 기록·완료일까지, 진행중이면 오늘까지 이어짐. 기록 없는 메모는 기한 자리에 점, 기간(~) 메모는 기간 그대로. 색 = 상태(노랑 할일, 초록 진행중, 회색 완료), 파란 세로줄 = 오늘. ‹ ›로 달 이동.
- **검색·업체 칩은 어느 보기에서든 동일** — 제목·진행기록·업체, 보관·완료까지 통합 검색. 보드에서 검색하면 걸린 보관 메모는 아래에 따로 표시되고, 표에서는 일치한 기록 줄이 날짜와 함께 표시.
- **백업 버튼** — 메모·점검·순서 전체를 JSON 파일로 내려받는다. 가끔 눌러서 파일을 보관해두면 계정 문제·실수 삭제에 대비한 보험이 된다 (이 파일로 복원 가능)

의도: "25년 ○○업체 계약건 내용 가져와줘" — 몇 년이 지나도 과정 전체가 그대로 나와야 한다. 판 전체를 조망할 땐 보드, 뒤질 땐 표, 일정 감각은 타임라인.

## 7. 점검 — 안전관리 점검 캘린더

회사 안전관리(법정 점검·선임·교육) 업무를 엑셀 대신 여기서 관리한다. 일반 메모와는 별도 공간.

- **연간 표**: 업무 × 1~12월. 시행하는 달에 O가 있고, 누르면 ✓(완료), 다시 누르면 취소. 시행 달이 지나도록 체크가 없으면 빨간 !로 바뀐다. 이번 달 열은 파란 배경.
- **행 수정·추가**: "+ 업무 추가"로 새 업무, 각 줄의 "수정"으로 분야·업무·주기·담당·증빙·시행 월(1~12 버튼)·★법정 필수(미이행 시 과태료) 표시를 언제든 바꿀 수 있다 — 엑셀에서 행 고치듯이.
- **수시·조건부**: 시행 월이 없는 업무(MSDS, 유해위험방지계획서 등)는 표 아래 별도 목록.
- **오늘 화면 연동**: 이번 달 시행 업무가 오늘 화면 "이번 달 점검" 섹션에 뜬다. 거기서 바로 완료 가능(표와 같은 데이터).
- **처음 시작**: 비어 있으면 "전임자 연간 캘린더 불러오기(22건)" 버튼으로 한 번에 채울 수 있다.
- **이력 패널**: 표의 줄(또는 오늘 화면의 점검 줄)을 누르면 메모와 똑같은 오른쪽 패널이 열린다. 진행 기록을 한 줄씩 남기고(수정·삭제·체크 가능), "보고서 수령 → 전기폴더 보관"처럼 증빙 위치도 여기 적는다. 기록이 있으면 표의 업무명 옆에 "기록 n" 표시. 월별 완료 체크는 표의 칸에서.
- **월별 이력**: 완료 체크한 달마다 패널에 이력 줄이 쌓인다 (`26년 7월 · 완료 07-04`). 줄의 "+ 특이사항"을 눌러 점검에서 나온 내용을 한 줄 기록 — 연도가 지나도 보존되어 "작년 6월에 뭐 나왔지?"에 답할 수 있다.
- **조치·수리 연결**: 점검에서 불량이 나오면 패널의 [+ 조치·수리 만들기] — 수리 메모가 생성되어(오늘 기한) 점검과 자동 연결된다. 수리는 여느 메모처럼 완료까지 괴롭히고, 진행기록·파일(견적서)·업체를 그대로 쓴다. 점검 패널에는 연결된 조치 목록이, 조치 메모에는 원래 점검으로 가는 링크가 표시된다.
- 패널이 열려 있는 동안은 표의 주기·담당·증빙 열이 잠시 숨어 12개월이 항상 다 보인다.
- 점검 탭의 빨간 숫자 = 이번 달 아직 안 한 업무 수.
- 증빙·계약서·견적서 파일은 이력 패널의 "파일" 칸에 올려두면 된다 — 해당 업무에 붙어서 어디서든 열람.
- **파일로 새 업무 등록**: 계약서·견적서 파일을 점검 화면에 끌어다 놓으면 새 행이 자동 생성된다 (업무명 = 파일명, 파일 첨부됨). 분야·시행 월만 바로 열리는 폼에서 지정.
- **분야별 정렬**: 상단 "분야별" 버튼으로 등록순 ↔ 분야별 전환.
- **보고서·엑셀**: 상단 [보고서] = 요약(이행률·과태료 리스크)과 분야별 이행 현황이 담긴 인쇄용 보고서 (인쇄 → PDF 저장, 표를 복사해 PPT에 붙여넣기 가능). [엑셀] = .xlsx 파일 다운로드.

## 로그인과 동기화

- 주소: https://vanguard-cell.github.io/memo-app/ — 회사·폰·집 어디서든 접속.
- 새 기기에서 처음 열면 로그인 화면이 나온다. **"Google로 로그인" 버튼 한 번**이면 끝 — 비밀번호 없음. (구글 계정이 없는 기기는 아래 "이메일 링크로 로그인" 사용)
- 브라우저 설정 등으로 로그인이 풀려도 버튼 한 번만 다시 누르면 된다.
- 어디서 적든 서버에 저장되어 모든 기기에서 같게 보인다. 인터넷이 끊겨도 기기에 저장되고, 화면 위에 "동기화 안 됨"이 뜨면 연결 후 다시 열면 된다.
- **자동 맞춤**: 탭이나 앱으로 다시 돌아오면(화면이 다시 보이면) 서버와 자동으로 맞춘다 — 폰에서 적고 회사 PC 탭으로 돌아오면 잠시 뒤 반영된다. 안 보이면 새로고침 한 번.

## 알아둘 것
- 상태는 자동: 할일(체크 없음) → 진행중(체크 하나라도 함) → 완료(완료 처리 버튼). 보드에서 드래그·버튼으로 직접 옮길 수도 있다. 보관은 던질 때 "보관함에 넣기"로만 들어간다.
- 정보 수정: 메모 패널 → "정보 수정"에서 제목·업체·기한·기간 수정 가능.
- 실행 방법: C:\HDS\memo-app 폴더에서 `npm run dev` → 브라우저에서 http://localhost:5173
