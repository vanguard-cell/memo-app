# 내 기록 (memo-app)

개인 업무기록 웹앱. 사용자가 회사·집·폰에서 쓰는 1인용 도구다. 상세한 사용법과 각 기능의 의도는 [사용설명서.md](사용설명서.md)를 먼저 읽을 것.

## 최우선 원칙: 심플함

- 새 기능을 넣을 때 "탭·버튼·개념을 늘리지 않고 되는가"를 먼저 물을 것. 추가보다 병합·제거를 우선.
- 화면 구성은 탭 3개(오늘·달력·메모) + 입력창 1개가 전부다. 이 틀을 늘리지 말 것.
- 팝업 금지 — 메모 상세는 오른쪽 사이드 패널. 자동 로그("완료 처리" 같은 것) 금지. 안내 문구 최소화.
- 사용자의 표현: "간단하지만 정확하게!"

## 핵심 컨셉 (사용자 요구사항)

1. 형식 없이 한 줄로 던지면 날짜·기간·업체를 자동 인식하되, 저장 전 칩으로 확인시킨다. 날짜는 제목에서 분리한다 ("7/6 이사" → 제목 "이사", 기한 7/6).
2. 같은 업체의 진행중 메모가 있으면 "이어붙이기"를 먼저 제안한다 — 하나의 사건은 하나의 메모에 (파편화 방지).
3. 기한이 지난 일은 완료할 때까지 오늘 화면에서 계속 괴롭힌다 ("3일째" 식으로). 계약 만기는 D-60부터 미리 알린다.
4. 진행 기록은 누적이며 체크박스로 완료 관리. 달력에는 뜨지 않고 메모 안에서만 산다.
5. 완료된 것도 절대 버리지 않는다 — "25년 ○○업체 계약건" 검색이 몇 년 뒤에도 돼야 한다.
6. 달력에는 모든 메모가 자기 날짜(기한·기간·작성일)에 표시된다. 드래그로 날짜 이동, 날짜별 목록에서 드래그로 우선순위 지정.

## 구조

- Vite + React (JS, TS 아님). 상태는 src/store.js (localStorage 키 `hds-memo-data-v1`, useSyncExternalStore).
- src/parser.js — 한국어 자연어 인식 (날짜·기간·업체). 정규식 기반.
- src/derive.js — 파생 계산 (상태, D-day, 오늘 화면의 괴롭히기 목록).
- src/views/ — TodayView(오늘), CalendarView(달력), MemosView(메모: 목록+검색+업체칩 통합).
- src/components/ — InputBar(던지기+이어붙이기), MemoDetail(사이드 패널), MemoRow.
- 상태는 자동 파생: 할일(기록 없음) → 진행중(기록 있음) → 완료. status 필드는 'open'/'done'만.
- store.js의 migrate()가 로드 시 옛 데이터를 현재 스키마로 변환한다. 스키마를 바꾸면 여기에 마이그레이션을 추가할 것.

  - 테이블: memos(id, user_id, data jsonb, updated_at), app_state(user_id, day_order). 둘 다 RLS로 본인 것만 접근.
  - 로그인: 구글 로그인(기본) + 이메일 매직링크(보조). 사용자 계정: vpfldh87@gmail.com. 구글 OAuth 클라이언트는 사용자의 Google Cloud "My First Project"에 있음.
  - 동기화: 로컬(localStorage) 즉시 반영 + 서버 upsert. 로그인 시 updatedAt 기준 병합. 실패 시 헤더에 "동기화 안 됨" 표시.
  - 배포 주소: https://vanguard-cell.github.io/memo-app/ (main 푸시 시 Actions 자동 배포).
  - 파일 첨부: Storage 버킷 'files' (비공개, 경로 {user_id}/{uuid}.{확장자} — 한글 파일명은 키로 못 쓰므로 uuid 경로 + 원본 이름은 메타데이터에). 메모·점검 공용 FileSection 컴포넌트 + 공용 업로드 헬퍼 src/files.js. files 배열은 data jsonb에 포함되어 함께 동기화.
  - 점검 탭: 파일 드롭 → 새 업무 행 자동 생성(업무명=파일명, 파일 첨부, 수정 폼 자동 오픈). 분야별 정렬 토글. 보고서(src/report.js — 인쇄용 HTML 새 창) + 엑셀(.xlsx, SheetJS) 내보내기.
- 3단계 (예정): PWA 폰 푸시 알림, 필요 시 멀티유저.

## 개발 메모

- 실행: `npm run dev` (포트 5173). 빌드: `npm run build`.
- **제거된 기능 — 다시 넣지 말 것**: 음성 입력(마이크 버튼, 2026-07-18 제거 — 브라우저 음성인식이 기기마다 오작동, 특히 아이폰 사파리 미지원). 사용자가 나중에 "회의록 녹음 → 정리" 기능을 원하니 그때 다른 방식(녹음 파일 + AI 정리)으로 설계할 것.
- 폰(<900px) 상세는 누른 줄 **바로 아래** 인라인으로 열려야 한다 (보드·표·타임라인·달력 공통). 목록 맨 밑에 렌더링하면 안 보이는 버그가 된다 (2026-07-18 수정). 폰 달력은 칸에 색 막대만 + **아래에 이 달 전체 날짜별 목록**(제목은 거기서, 오늘 기본 선택, 칸 탭 → 목록 스크롤). 기간 중간 날짜는 목록에서 생략(날짜 전용 기록 있을 때만). 좁은 화면 판정은 src/useIsNarrow.js 공용 훅.
- 사용자는 개발자가 아니다. 설명은 쉬운 한국어로, 기술 용어 최소화.
- 기능을 바꾸면 사용설명서.md도 같이 갱신할 것.
